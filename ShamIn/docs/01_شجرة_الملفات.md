# شجرة ملفات مشروع ShamIn — شرح تفصيلي لكل ملف

## نظرة عامة

مشروع ShamIn هو نظام ذكي لتوقع سعر صرف الليرة السورية مقابل الدولار الأمريكي. يتكون المشروع من 81 ملفاً موزعة على 33 مجلداً، مُنظمة بنمط معماري متعدد الطبقات (Layered Architecture).

---

## الشجرة الكاملة مع الشرح

```
ShamIn/                              ← المجلد الجذري للمشروع
│
├── .env.example                     ← ملف نموذجي لمتغيرات البيئة (لا يُرفع إلى Git)
├── .gitignore                       ← قائمة الملفات المُستثناة من Git
├── docker-compose.yml               ← تعريف خدمات Docker (قواعد البيانات + طابور المهام)
├── Dockerfile                       ← تعريف حاوية Docker للتطبيق
├── README.md                        ← ملف التعريف بالمشروع
├── requirements.txt                 ← قائمة مكتبات Python المطلوبة
```

### مجلد config/ — ملفات الإعدادات

```
├── config/
│   ├── settings.yaml                ← الإعدادات العامة للمشروع
│   ├── sources.yaml                 ← تعريف مصادر البيانات (مواقع، تلغرام، RSS)
│   ├── model_config.yaml            ← إعدادات نماذج التعلم الآلي
│   └── alerts.yaml                  ← قواعد التنبيهات والإشعارات
```

**شرح تفصيلي:**

| الملف | الدور | ما يحتويه |
|-------|-------|----------|
| `settings.yaml` | الإعدادات العامة | اسم المشروع، إصداره، روابط قواعد البيانات، إعدادات التسجيل (Logging)، إعدادات المراقبة |
| `sources.yaml` | تعريف المصادر | قائمة مواقع الأسعار (sp-today، investing.com)، قنوات تلغرام (أسعار + أخبار)، خلاصات RSS (عنب بلدي، رويترز، سانا)، واجهات API خارجية (ذهب، نفط، DXY) |
| `model_config.yaml` | إعدادات النماذج | إعدادات TFT (الطبقات المخفية، معدل التسرب)، إعدادات XGBoost، إعدادات LSTM للمشاعر، تعريف الميزات (Features)، إعدادات التدريب والتقييم |
| `alerts.yaml` | قواعد التنبيه | عتبات التغير الحاد (>3%)، خطأ التنبؤ (MAPE>5%)، تعطل المصادر، انجراف البيانات، تدهور الأداء |

### مجلد src/ — الكود الأساسي

#### src/ingestion/ — طبقة جمع البيانات

```
├── src/
│   ├── __init__.py                  ← يجعل src حزمة Python قابلة للاستيراد
│   │
│   ├── ingestion/                   ← طبقة جمع البيانات من المصادر المختلفة
│   │   ├── __init__.py
│   │   ├── scheduler.py             ← مُجدوِل المهام (Celery Beat) — يُحدد متى يتم الجمع
│   │   ├── raw_schema.py            ← مخططات Pydantic للتنسيق الموحد للبيانات
│   │   └── collectors/              ← الجامعات (Collectors)
│   │       ├── __init__.py
│   │       ├── base.py              ← الكلاس الأساسي لكل الجامعات (BaseCollector)
│   │       ├── rss_collector.py     ← جامع خلاصات RSS (أخبار عنب بلدي، رويترز، سانا)
│   │       ├── telegram_collector.py ← جامع قنوات تلغرام (أسعار + أخبار)
│   │       ├── web_scraper.py       ← كاشط مواقع الويب (sp-today، investing.com)
│   │       └── api_collector.py     ← جامع واجهات API الخارجية (ذهب، نفط، DXY)
```

**شرح تفصيلي:**

| الملف | الدور الدقيق |
|-------|-------------|
| `base.py` | يُعرِّف الكلاس المجرد `BaseCollector` الذي تَرِثه جميع الجامعات. يوفر: طريقة حساب التجزئة SHA-256 لمنع التكرار، تحويل البيانات إلى تنسيق موحد، إعادة المحاولة مع تأخير تصاعدي (Exponential Backoff) |
| `rss_collector.py` | يستخدم مكتبة `feedparser` لجمع الأخبار من خلاصات RSS. يعمل بشكل غير متزامن (async). يُستخرَج العنوان والملخص والرابط والمؤلف |
| `telegram_collector.py` | يستخدم مكتبة `Telethon` للاتصال بقنوات تلغرام. يدعم: تعدد الحسابات (Session Pool)، تأخيرات عشوائية لتجنب الحظر، معالجة خطأ FloodWait |
| `web_scraper.py` | يستخدم `aiohttp` و `BeautifulSoup` لاستخراج أسعار الصرف من المواقع. يدعم CSS Selector و Regex |
| `api_collector.py` | يجمع المؤشرات الاقتصادية الخارجية (سعر الذهب، النفط، مؤشر الدولار) من واجهات API |
| `scheduler.py` | يُعرِّف تطبيق Celery مع 5 مهام مجدولة: RSS (كل 15 دقيقة)، تلغرام أسعار (كل 5 دقائق)، تلغرام أخبار (كل 5 دقائق)، كشط مواقع (كل 5 دقائق)، مؤشرات خارجية (كل ساعة) |
| `raw_schema.py` | يُعرِّف مخططي Pydantic: `RawDataItem` (البيانات الخام الموحدة) و `CollectionResult` (نتيجة عملية الجمع) |

#### src/processing/ — طبقة المعالجة

```
│   ├── processing/                  ← طبقة معالجة البيانات الخام
│   │   ├── __init__.py
│   │   ├── pipeline.py              ← خط الأنابيب الرئيسي (يُنسّق التنظيف والاستخراج)
│   │   ├── text/                    ← معالجة النصوص
│   │   │   ├── __init__.py
│   │   │   ├── cleaner.py           ← مُنظِّف النصوص العربية
│   │   │   ├── tokenizer.py         ← مُقطِّع النصوص العربية (Tokenizer)
│   │   │   ├── classifier.py        ← مُصنِّف الأحداث (عسكري/سياسي/اقتصادي/اجتماعي)
│   │   │   ├── sentiment.py         ← مُحلِّل المشاعر (إيجابي/سلبي/محايد)
│   │   │   └── embeddings.py        ← تمثيلات الكلمات (Word2Vec/GloVe/FastText)
│   │   ├── numeric/                 ← المعالجة الرقمية
│   │   │   ├── __init__.py
│   │   │   ├── extractor.py         ← مُستخرِج الأسعار والنسب من النصوص
│   │   │   └── interpolation.py     ← استيفاء الفجوات الزمنية في البيانات
│   │   └── features/                ← هندسة الميزات
│   │       ├── __init__.py
│   │       ├── engineer.py          ← حساب المؤشرات الفنية (MA, RSI, MACD, Bollinger)
│   │       ├── weights.py           ← الأوزان الديناميكية للأحداث مع اضمحلال أسي
│   │       ├── aggregator.py        ← تجميع الميزات في نوافذ زمنية
│   │       ├── selector.py          ← اختيار الميزات بالمعلومات المتبادلة (Mutual Information)
│   │       └── store.py             ← متجر الميزات المركزي (Feature Store)
```

**شرح تفصيلي:**

| الملف | الدور الدقيق |
|-------|-------------|
| `pipeline.py` | يُنشئ خط أنابيب معالجة يربط بين `TextCleaner` و `NumericExtractor`. دالة `process()` تأخذ عنصر بيانات خام وتُخرج عنصراً معالجاً. دالة `process_batch()` تعالج مجموعة عناصر |
| `cleaner.py` | يُنظِّف النصوص العربية عبر 8 خطوات: إزالة HTML، إزالة URLs، إزالة mentions/hashtags، إزالة Emojis، تطبيع الأحرف العربية (أ/إ/آ→ا)، تحويل الأرقام الشرقية للغربية، إزالة التشكيل، تطبيع المسافات |
| `tokenizer.py` | يُقطِّع النصوص إلى كلمات مع إزالة كلمات التوقف (Stop Words). يدعم تكامل Farasa الاختياري |
| `extractor.py` | يستخرج الأسعار بالليرة (أنماط regex متعددة)، النسب المئوية، واتجاه السعر (ارتفاع/انخفاض/استقرار) من النصوص العربية |
| `classifier.py` | (مرحلة 2.3) تصنيف الأخبار إلى 4 فئات: عسكري، سياسي، اقتصادي، اجتماعي |
| `sentiment.py` | (مرحلة 2.4) تحليل مشاعر الأخبار: إيجابي، سلبي، محايد |
| `embeddings.py` | (مرحلة 2.5) تحويل الكلمات إلى متجهات رقمية |
| `engineer.py` | (مرحلة 3) حساب المؤشرات الفنية: المتوسطات المتحركة، RSI، MACD، بولنجر، ATR |
| `weights.py` | (مرحلة 3) حساب أوزان ديناميكية للأحداث مع اضمحلال أسي عبر الزمن |
| `aggregator.py` | (مرحلة 3) تجميع الميزات في نوافذ زمنية (1 ساعة، 6 ساعات، 24 ساعة) |
| `selector.py` | (مرحلة 3) اختيار أهم الميزات باستخدام المعلومات المتبادلة |
| `store.py` | (مرحلة 3) إدارة مركزية لجميع الميزات المحسوبة |

#### src/prediction/ — طبقة النماذج والتنبؤ

```
│   ├── prediction/                  ← طبقة نماذج التعلم الآلي
│   │   ├── __init__.py
│   │   ├── feedback.py              ← حلقة التغذية الراجعة (إعادة التدريب التلقائية)
│   │   ├── models/                  ← تعريفات النماذج
│   │   │   ├── __init__.py
│   │   │   ├── baseline.py          ← نماذج خط الأساس (XGBoost / RandomForest)
│   │   │   ├── time_series.py       ← نماذج السلاسل الزمنية (HLT / SES / ARIMA)
│   │   │   ├── tft_model.py         ← نموذج TFT المخصص (النموذج الرئيسي)
│   │   │   └── deep_learning.py     ← نماذج التعلم العميق للمشاعر (CNN/RNN/LSTM)
│   │   ├── training/                ← أدوات التدريب
│   │   │   ├── __init__.py
│   │   │   ├── trainer.py           ← المُدرِّب (PyTorch Lightning Trainer)
│   │   │   ├── evaluator.py         ← المُقيِّم (MAE, RMSE, MAPE, دقة الاتجاه)
│   │   │   ├── backtester.py        ← الاختبار الخلفي (Backtesting)
│   │   │   └── optimizer.py         ← مُحسِّن المعاملات الفائقة (Optuna)
│   │   └── inference/               ← أدوات الاستدلال
│   │       ├── __init__.py
│   │       ├── predictor.py         ← المُتنبئ الهجين (يجمع جميع النماذج)
│   │       └── explainer.py         ← المُفسِّر (SHAP / Grad-CAM)
```

**شرح تفصيلي:**

| الملف | الدور الدقيق |
|-------|-------------|
| `baseline.py` | (مرحلة 4) نماذج خط الأساس: XGBoost و RandomForest. تُدرَّب على الميزات المسطحة. تُعطي أداءً مرجعياً لمقارنة النماذج الأكثر تعقيداً |
| `time_series.py` | (مرحلة 5) نماذج السلاسل الزمنية الكلاسيكية: HLT (Holt's Linear Trend)، SES (Simple Exponential Smoothing)، ARIMA |
| `tft_model.py` | (مرحلة 6) النموذج الرئيسي: Temporal Fusion Transformer. يُعالج سلاسل زمنية مع متغيرات معروفة ومجهولة، ثابتة ومتغيرة. يُخرج تنبؤات كمية (Quantile Predictions) |
| `deep_learning.py` | (مرحلة 7) شبكات عصبية للمشاعر: LSTM ثنائي الاتجاه يعمل على تمثيلات الكلمات |
| `trainer.py` | (مرحلة 6) يُغلِّف PyTorch Lightning Trainer مع: تقدير حجم الدفعة تلقائياً، Early Stopping، حفظ أفضل النقاط (Checkpointing) |
| `evaluator.py` | (مرحلة 6) حساب مقاييس الأداء: MAE، RMSE، MAPE، R²، دقة اتجاه السعر |
| `backtester.py` | (مرحلة 6) اختبار النموذج على بيانات تاريخية بنوافذ متحركة (Walk-Forward Validation) |
| `optimizer.py` | (مرحلة 6) ضبط المعاملات الفائقة تلقائياً باستخدام Optuna (حجم الطبقات، معدل التعلم، معدل التسرب) |
| `predictor.py` | (مرحلة 8) يُجمع تنبؤات جميع النماذج بتصويت مرجح (Weighted Voting) أو تكديس (Stacking) |
| `explainer.py` | (مرحلة 8) يشرح تنبؤات النموذج: أي ميزات أثرت أكثر (SHAP)، أي أحداث أخبارية |
| `feedback.py` | (مرحلة 9) يُراقب أداء النموذج ويُطلق إعادة تدريب عند تدهور الدقة |

#### src/storage/ — طبقة التخزين

```
│   ├── storage/                     ← طبقة التخزين وقواعد البيانات
│   │   ├── __init__.py
│   │   ├── relational_db.py         ← عميل PostgreSQL (SQLAlchemy)
│   │   ├── timeseries_db.py         ← عميل InfluxDB (سلاسل زمنية)
│   │   ├── object_store.py          ← عميل MinIO (تخزين كائنات — نماذج، تقارير)
│   │   └── model_registry.py        ← سجل إصدارات النماذج
```

**شرح تفصيلي:**

| الملف | الدور الدقيق |
|-------|-------------|
| `relational_db.py` | يوفر دالتين: `get_engine()` لإنشاء محرك SQLAlchemy متصل بـ PostgreSQL، و `get_session()` لإنشاء جلسة قاعدة بيانات. يستخدم Connection Pool بحجم 10 |
| `timeseries_db.py` | كلاس `TimeSeriesDB` يتصل بـ InfluxDB. يوفر: `write_price()` لكتابة نقطة سعر، `query_prices()` لاستعلام أسعار فترة معينة باستخدام لغة Flux |
| `object_store.py` | كلاس `ObjectStore` يتصل بـ MinIO. يوفر: `save_model()` لحفظ نموذج مُدرَّب، `load_model()` لتحميله، `save_json()` لحفظ بيانات JSON، `list_models()` لسرد النماذج المحفوظة |
| `model_registry.py` | كلاس `ModelRegistry` يُدير سجل إصدارات النماذج محلياً (ملف JSON). يوفر: `register()` لتسجيل إصدار جديد مع مقاييس أدائه، `set_active()` لتحديد النموذج النشط، `get_active()` للحصول على النموذج النشط |

### مجلدات البيانات والدعم
