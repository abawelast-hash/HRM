# 1. شجرة ملفات مشروع ShamIn

## نظرة عامة

مشروع ShamIn هو نظام ذكي لتوقع سعر صرف الليرة السورية مقابل الدولار الأمريكي. يتكون المشروع من 81 ملفاً موزعة على 33 مجلداً، مُنظمة بنمط معماري متعدد الطبقات (Layered Architecture).

---

## الشجرة الكاملة مع الشرح

```
ShamIn/                              ← المجلد الجذري للمشروع
│
├── .env.example                     ← ملف نموذجي لمتغيرات البيئة (لا يُرفع إلى Git)
├── .gitignore                       ← قائمة الملفات المُستثناة من Git
├── docker-compose.yml               ← تعريف خدمات Docker (قواعد البيانات + طابور المهام)
├── Dockerfile                       ← تعريف حاوية Docker للتطبيق
├── README.md                        ← ملف التعريف بالمشروع
├── requirements.txt                 ← قائمة مكتبات Python المطلوبة
```

### مجلد config/ — ملفات الإعدادات

```
├── config/
│   ├── settings.yaml                ← الإعدادات العامة للمشروع
```

| `model_config.yaml` | إعدادات النماذج | إعدادات TFT، XGBoost، LSTM، تعريف الميزات، إعدادات التدريب |

# 2. مرجع المتغيرات والدوال والكلاسات

## src/utils/config.py — مُحمِّل الإعدادات

### المتغيرات العامة

| المتغير | النوع | الوصف |
|---------|-------|-------|
| `_CONFIG_DIR` | `Path` | المسار المطلق لمجلد `config/` |

### الدوال

#### `_resolve_env_vars(value)`
- **المُدخل**: أي قيمة (نص، قاموس، قائمة)
- **المُخرج**: نفس القيمة مع استبدال `${VAR}` بقيم متغيرات البيئة
- **الشرح**: تعمل بشكل عودي (Recursive)

#### `load_config(filename: str) -> dict`
- يفتح ملف YAML من `_CONFIG_DIR`، يقرأه بـ `yaml.safe_load`، ثم يُمرره عبر `_resolve_env_vars`

#### `get_settings()` / `get_sources()` / `get_model_config()` / `get_alerts_config()`
- تُرجع محتوى ملفات الإعدادات الأربعة

## src/utils/validation_utils.py

#### `validate_dataframe(df, required_columns, name="DataFrame")`

# 4. مسار البيانات — من الإدخال إلى المخرجات

## المخطط العام

```
مصادر (تلغرام/RSS/ويب/API)
        ↓
تنسيق موحد (id, source, timestamp, raw_text, content_hash)
        ↓
معالجة (تنظيف → استخراج أرقام → مشاعر → تصنيف)
        ↓
تخزين (PostgreSQL + InfluxDB + MinIO)
        ↓
هندسة ميزات (17 ميزة زمنية + 7 تصنيفية)
        ↓
نمذجة (TFT 60% + XGBoost 25% + HLT 15%)
        ↓
مخرجات (API + Dashboard + تنبيهات)
```

---

## آليات الإدخال الأربع

### 1. تلغرام (كل 5 دقائق)
- يقرأ آخر 100 رسالة من قنوات أسعار/أخبار
- يضيف تأخير عشوائي (0.5-2 ث) لتجنب الحظر
- **التأثير**: أسعار فورية + مشاعر سوق

### 2. RSS (كل 15 دقيقة)
- يجمع أخبار من مواقع عربية (عنب بلدي، الدرر الشامية...)
- **التأثير**: تُصنف كأحداث وتُؤثر في الميزات

### 3. كشط ويب (كل 5 دقائق)
- يستخرج أسعار من مواقع بـ CSS Selector / Regex
- **التأثير**: بيانات أسعار دقيقة مباشرة

### 4. APIs خارجية (كل ساعة)
- مؤشرات: ذهب (XAU_USD)، نفط (BRENT)، دولار (DXY)، ليرة تركية، ليرة لبنانية
- **التأثير**: متغيرات خارجية (exogenous) في TFT

---

## خط المعالجة

1. **تنظيف النص** (8 خطوات): HTML → URLs → Mentions → Emojis → تطبيع عربي → أرقام → تشكيل → مسافات
2. **استخراج أرقام**: سعر (3 أنماط regex)، نسبة، اتجاه (up/down/stable)
3. **تحليل مشاعر** (مرحلة لاحقة): positive/negative/neutral مع درجة -1 إلى 1
4. **تصنيف أحداث** (مرحلة لاحقة): military/political/economic/social مع وزن 0-1

---

## الميزات المُدخلة للنموذج

**17 ميزة متغيرة غير معروفة مسبقاً**: syp_usd_price، price_change_pct، price_volatility، volume_proxy، sentiment_score، أوزان أحداث (عسكري/سياسي/اقتصادي/اجتماعي)، ذهب، نفط، DXY، ليرة تركية، ليرة لبنانية، عدد أخبار/ساعة، embedding dimensions

**7 ميزات تصنيفية معروفة مسبقاً**: ساعة اليوم، يوم الأسبوع، عطلة، شهر، عطلة سورية، عطلة تركية، رمضان

---

## التنبؤ

```
التنبؤ_النهائي = 0.60 × TFT + 0.25 × XGBoost + 0.15 × HLT
```

المخرج يحتوي: سعر متوقع، اتجاه، ثقة، 5 كميات، أهم الميزات، الأحداث المؤثرة

---

## مثال عملي

```
خبر: "قصف عنيف في دمشق"
  → تنظيف → "قصف عنيف في دمشق"
  → استخراج → {price: None, direction: None}
  → مشاعر → {sentiment: "negative", score: -0.8}
  → تصنيف → {category: "military", weight: 0.9}
  → ميزة → military_event_weight = 0.9
  → TFT → price↑ (أحداث عسكرية ← عدم استقرار ← ارتفاع دولار)
  → Dashboard → "⚠️ توقع ارتفاع إلى 14800 خلال 24 ساعة"
```
