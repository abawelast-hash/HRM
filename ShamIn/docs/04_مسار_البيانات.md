# مسار البيانات — من الإدخال إلى المخرجات

## نظرة عامة

هذا الملف يشرح رحلة البيانات الكاملة في مشروع ShamIn: من أين تأتي البيانات، كيف تُعالج، وكيف تؤثر في التنبؤ بسعر الصرف.

## المرحلة 1: آليات الإدخال

### 1.1 جمع من تلغرام (`TelegramCollector`)

**المصدر**: قنوات تلغرام سورية تنشر أسعار الصرف والأخبار الاقتصادية

**آلية العمل**:
1. يتصل بـ Telegram API عبر حسابات متعددة (لتجنب الحظر)
2. يقرأ آخر 100 رسالة من كل قناة خلال آخر 24 ساعة
3. يتخطى الرسائل بدون نص
4. يُضيف تأخيراً عشوائياً (0.5-2 ثانية) بين الرسائل لتجنب الحظر
5. عند تلقي `FloodWaitError` ينتقل لحساب آخر

**الجدولة**: كل 5 دقائق

**البيانات الناتجة**:
```python
{
    "raw_text": "الدولار مقابل الليرة السورية: 14500 ل.س",
    "timestamp": "2024-01-15 10:30:00",
    "metadata": {"channel": "@syrian_rates", "message_id": 12345}
}
```

**التأثير على المخرجات**: النص يُحلل لاستخراج أسعار فورية ومشاعر السوق

---

### 1.2 جمع من RSS (`RSSCollector`)

**المصدر**: مواقع أخبار عربية (عنب بلدي، الدرر الشامية، السورية نت، المرصد...)

**آلية العمل**:
1. يُحمِّل خلاصة RSS بـ `feedparser`
2. يقرأ العنوان والملخص والرابط وتاريخ النشر
3. يُنشئ بصمة SHA-256 لمنع التكرار

**الجدولة**: كل 15 دقيقة

**البيانات الناتجة**:
```python
{
    "raw_text": "العنوان: ... | الملخص: ...",
    "metadata": {"url": "https://...", "author": "..."}
}
```

**التأثير على المخرجات**: الأخبار تُصنف كأحداث (عسكرية، سياسية، اقتصادية) وتُؤثر في الميزات

---

### 1.3 كشط الويب (`WebScraper`)

**المصدر**: مواقع أسعار صرف سورية تُحدِّث الأسعار أكثر من RSS

**آلية العمل**:
1. يُحمِّل صفحة HTML بـ `aiohttp`
2. يستخدم CSS Selector أو Regex لاستخراج السعر
3. يُحلل السعر ويُنظفه من الفواصل والعملات

**الجدولة**: كل 5 دقائق

**البيانات الناتجة**:
```python
{
    "raw_text": "",
    "raw_numeric": 14500.0,
    "metadata": {"url": "https://..."}
}
```

**التأثير على المخرجات**: يوفر بيانات أسعار دقيقة مباشرة (لا تحتاج استخراج من نص)

---

### 1.4 جمع من APIs خارجية (`APICollector`)

**المصدر**: واجهات برمجة لمؤشرات اقتصادية عالمية

**آلية العمل**:
1. يُرسل طلب GET لـ API
2. يُحلل الاستجابة JSON
3. يستخرج السعر/القيمة

**الجدولة**: كل ساعة

**المؤشرات المجمعة** (من `config/sources.yaml`):
- `XAU_USD` — سعر الذهب بالدولار
- `BRENT` — سعر برميل النفط
- `DXY` — مؤشر قوة الدولار
- `TRY_USD` — سعر الليرة التركية (اقتصاد مرتبط بسوريا)
- `LBP_USD` — سعر الليرة اللبنانية (اقتصاد مرتبط)
- `CPI_SY` — مؤشر أسعار المستهلك السوري

**التأثير على المخرجات**: هذه المؤشرات تُستخدم كمتغيرات خارجية (exogenous variables) في نموذج TFT — ارتفاع الذهب أو النفط يؤثر على سعر الليرة

## المرحلة 2: التوحيد

كل البيانات من المصادر الأربعة تُحوَّل إلى **التنسيق الموحد** عبر `BaseCollector.to_unified_format()`:

```python
{
    "id": "uuid4...",            # معرف فريد عالمي
    "source": "enab_baladi",     # اسم المصدر
    "source_type": "rss",        # النوع
    "timestamp": datetime(...),   # وقت البيانات الأصلي
    "raw_text": "...",           # النص الخام
    "raw_numeric": 14500.0,      # رقم (إن وُجد)
    "language": "ar",            # اللغة
    "metadata": {...},           # بيانات إضافية
    "content_hash": "sha256..."  # بصمة فريدة لمنع التكرار
}
```

**الغرض**: توحيد من مصادر متعددة ← مدخل واحد لخط المعالجة

## المرحلة 3: المعالجة

### 3.1 تنظيف النص (`TextCleaner.clean()`)

**المدخل**: نص عربي خام من المصدر
**المخرج**: نص مُنظف جاهز للتحليل

**الخطوات الثمان بالترتيب**:

| الخطوة | الدالة | مثال المدخل | مثال المخرج |
|--------|--------|------------|------------|
| 1 | `remove_html()` | `<b>الدولار</b>` | `الدولار` |
| 2 | `remove_urls()` | `زوروا https://ex.com` | `زوروا` |
| 3 | `remove_mentions_hashtags()` | `@أحمد #اقتصاد` | `` |
| 4 | `remove_emojis()` | `ارتفاع 📈` | `ارتفاع` |
| 5 | `normalize_arabic()` | `إبراهيم الأمة` | `ابراهيم الامه` |
| 6 | `convert_numerals()` | `١٤٥٠٠` | `14500` |
| 7 | `remove_diacritics()` | `الدَّوْلارُ` | `الدولار` |
| 8 | `normalize_whitespace()` | `الدولار   اليوم` | `الدولار اليوم` |

**التأثير**: النص المُنظف أدق في تحليل المشاعر واستخراج الأرقام

---

### 3.2 استخراج الأرقام (`NumericExtractor.extract_all()`)

**المدخل**: نص مُنظف
**المخرج**: قاموس `{price, percentage, direction}`

**كيف يُستخرج السعر**:
1. يبحث عن نمط: `رقم + ل.س/ليرة/SYP` ← مثل "14500 ل.س"
2. إذا فشل، يبحث عن: `الدولار + رقم` ← مثل "الدولار 14500"
3. إذا فشل، يبحث عن أي رقم من 4-7 خانات (مثل 14500)
4. يتحقق أن السعر بين 100 و 1,000,000

**كيف يُستخرج الاتجاه**:
- كلمات ↑: ارتفع، صعود، زيادة → `"up"`
- كلمات ↓: انخفض، هبوط، تراجع → `"down"`
- كلمات ↔: استقر، ثبات → `"stable"`

**التأثير**: السعر المستخرج يُضاف مباشرة لسلسلة الأسعار الزمنية

---

### 3.3 تحليل المشاعر (المرحلة 3 — لم يُنفَّذ بعد)

**المدخل**: نص مُنظف
**المخرج**: `{sentiment: "positive/negative/neutral", score: -1.0 to 1.0}`

**كيف سيعمل**:
1. تقطيع النص بـ `ArabicTokenizer`
2. تمريره عبر نموذج LSTM مُدرب على بيانات عربية
3. الحصول على درجة المشاعر

**التأثير على التنبؤ**: المشاعر الإيجابية (استقرار، اتفاقيات) ← تميل لانخفاض الدولار. المشاعر السلبية (حرب، عقوبات) ← تميل لارتفاع الدولار

---

### 3.4 تصنيف الأحداث (المرحلة 4 — لم يُنفَّذ بعد)

**المدخل**: نص مُنظف
**المخرج**: `{category: "military/political/economic/social", weight: 0.0-1.0}`

**الفئات وتأثيرها**:

| الفئة | أمثلة | التأثير المتوقع |
|-------|-------|----------------|
| عسكري (military) | قصف، اشتباك، هجوم | ↑ سعر الدولار (عدم استقرار) |
| سياسي (political) | عقوبات، اتفاق، مفاوضات | متغير حسب السياق |
| اقتصادي (economic) | رواتب، تضخم، استيراد | تأثير مباشر على العرض والطلب |
| اجتماعي (social) | احتجاجات، أزمات | ↑ سعر الدولار (قلق شعبي) |

**الوزن**: حدث عسكري كبير (وزن 0.9) يؤثر أكثر من خبر اجتماعي هامشي (وزن 0.2)

**اضمحلال الوزن**: تأثير الحدث يتناقص بمرور الوقت (اضمحلال أسي) — خبر عمره أسبوع أقل تأثيراً من خبر عمره ساعة

---

### 3.5 توليد التمثيل الرقمي (Embeddings — المرحلة 5)

**المدخل**: نص مُنظف
**المخرج**: متجه رقمي (vector) بأبعاد ثابتة يمثل معنى النص

**التأثير**: يسمح للنموذج بفهم المعنى الدلالي للأخبار (أخبار متشابهة ← متجهات قريبة)

## المرحلة 5: هندسة الميزات

### الميزات المُدخلة لنموذج TFT (من `config/model_config.yaml`):

#### ميزات متغيرة غير معروفة مسبقاً (`time_varying_unknown_reals`):
هذه ميزات لا نعرف قيمتها المستقبلية — النموذج يتعلم تأثيرها:

| الميزة | الوصف | التأثير على التنبؤ |
|--------|-------|--------------------|
| `syp_usd_price` | سعر الصرف الحالي | **الأهم** — السعر السابق يُنبئ بالسعر القادم |
| `price_change_pct` | نسبة التغير | تسارع/تباطؤ الحركة |
| `price_volatility` | التقلب | تقلب عالٍ ← عدم يقين ← توقعات أوسع |
| `volume_proxy` | حجم التداول التقريبي | صفقات كثيرة ← ضغط سعري |
| `sentiment_score` | درجة المشاعر | إيجابي ← انخفاض دولار، سلبي ← ارتفاع |
| `military_event_weight` | وزن الأحداث العسكرية | أحداث عسكرية ← ارتفاع الدولار |
| `political_event_weight` | وزن الأحداث السياسية | عقوبات ← ارتفاع، مفاوضات ← انخفاض |
| `economic_event_weight` | وزن الأحداث الاقتصادية | تأثير مباشر على العرض والطلب |
| `social_event_weight` | وزن الأحداث الاجتماعية | احتجاجات ← قلق ← ارتفاع |
| `gold_price_usd` | سعر الذهب | الذهب ملاذ آمن — ارتفاعه يعني عدم ثقة |
| `oil_price_brent` | سعر النفط | يؤثر على اقتصاد المنطقة |
| `usd_index_dxy` | مؤشر الدولار العالمي | دولار قوي عالمياً ← يرتفع محلياً |
| `try_usd` | الليرة التركية | اقتصاد مرتبط — تركيا شريك تجاري |
| `lbp_usd` | الليرة اللبنانية | اقتصاد مرتبط — حركة رؤوس أموال |
| `news_count_1h` | عدد الأخبار في ساعة | أخبار كثيفة ← حدث مهم |
| `embedding_dim_0` إلى `embedding_dim_63` | أبعاد التمثيل | المحتوى الدلالي للأخبار |

#### ميزات متغيرة معروفة مسبقاً (`time_varying_known_categoricals`):
هذه ميزات نعرف قيمتها المستقبلية — تساعد النموذج على فهم الأنماط:

| الميزة | الوصف | التأثير |
|--------|-------|---------|
| `hour_of_day` | ساعة اليوم (0-23) | أسواق الصرف أنشط في ساعات محددة |
| `day_of_week` | يوم الأسبوع (0-6) | الجمعة/السبت عطلة ← نشاط أقل |
| `is_weekend` | هل هو عطلة | حركة أقل في العطلة |
| `month` | الشهر (1-12) | أنماط موسمية (رمضان، أعياد) |
| `is_holiday_sy` | عطلة سورية | تأثير على الأسواق |
| `is_holiday_tr` | عطلة تركية | تأثير على التجارة |
| `is_ramadan` | شهر رمضان | زيادة الطلب على الدولار |

## المرحلة 6: النمذجة والتنبؤ

### كيف تعمل النماذج الثلاثة:

#### 6.1 نموذج TFT (الرئيسي — وزن 60%)
- **المدخل**: نافذة 168 ساعة (أسبوع) من كل الميزات
- **المخرج**: تنبؤ لـ 24 أو 72 ساعة بـ 5 كميات (5%, 25%, 50%, 75%, 95%)
- **الميزة الفريدة**: Interpretable Multi-Horizon Attention — يُخبرك أي ميزة أثرت أكثر
- **مثال مخرج**: "السعر بعد 24 ساعة: 14600 (ثقة 50%)، نطاق [14200-15000] (ثقة 90%)"

#### 6.2 نموذج XGBoost (مساعد — وزن 25%)
- **المدخل**: نفس الميزات لكن بشكل جدولي (صف واحد)
- **المخرج**: سعر واحد متوقع
- **الميزة**: سريع جداً في التدريب والاستدلال

#### 6.3 نموذج Holt-Winters (أساسي — وزن 15%)
- **المدخل**: سلسلة أسعار فقط (بدون ميزات إضافية)
- **المخرج**: سعر واحد متوقع بناءً على الاتجاه والموسمية
- **الميزة**: لا يحتاج ميزات — يعمل على السعر وحده

#### 6.4 التجميع الهجين (Hybrid Ensemble)
```
التنبؤ_النهائي = 0.60 × TFT + 0.25 × XGBoost + 0.15 × HLT
```
- الأوزان ديناميكية — تتغير حسب أداء كل نموذج في آخر فترة
- إذا كان TFT يخطئ كثيراً، ينخفض وزنه ويزيد وزن الآخرين

## المرحلة 7: المخرجات

### 7.1 التنبؤات (PostgreSQL → `predictions`)

كل تنبؤ يحتوي:
```python
{
    "predicted_price": 14600.0,       # السعر المتوقع
    "predicted_direction": "up",      # الاتجاه: ارتفاع
    "confidence": 0.85,               # مستوى الثقة: 85%
    "horizon_hours": 24,              # بعد 24 ساعة
    "q05": 14100, "q25": 14300,       # الحدود الدنيا
    "q50": 14600, "q75": 14900,       # الوسيط والربع الثالث
    "q95": 15200,                     # الحد الأقصى
    "top_features": [                 # أهم الميزات المؤثرة
        {"name": "military_event_weight", "importance": 0.23},
        {"name": "gold_price_usd", "importance": 0.18},
        ...
    ],
    "contributing_events": [          # الأحداث المؤثرة
        {"text": "قصف في...", "category": "military", "weight": 0.9}
    ]
}
```

### 7.2 لوحة المراقبة (Streamlit Dashboard)

4 صفحات:
1. **التنبؤات**: رسم بياني للسعر الحالي والمتوقع مع نطاقات الثقة
2. **الأحداث**: آخر الأحداث المصنفة وتأثيرها على السعر
3. **أداء النماذج**: مقارنة MAE/RMSE/MAPE بين النماذج عبر الزمن
4. **التنبيهات**: إشعارات عند تجاوز عتبات محددة

### 7.3 واجهة API (FastAPI)

نقاط النهاية:
- `GET /` — حالة التطبيق
- `GET /health` — فحص الصحة
- `GET /predictions/latest` — آخر تنبؤ (سيُضاف في مرحلة لاحقة)
- `POST /predictions/custom` — تنبؤ مخصص (سيُضاف)

### 7.4 نظام التنبيهات (Alerts)

يُرسل تنبيهات عند:
- تغير السعر أكثر من 5% في ساعة
- اكتشاف حدث عسكري بوزن > 0.8
- انحراف النموذج (Drift) عن الأداء الطبيعي
- فشل عملية الجمع 3 مرات متتالية

## ملخص: كيف يؤثر كل مدخل على المخرجات

```
خبر تلغرام: "قصف عنيف في دمشق" 
    ↓
TextCleaner → "قصف عنيف في دمشق"
    ↓
NumericExtractor → {price: None, direction: None}
    ↓
SentimentAnalyzer → {sentiment: "negative", score: -0.8}
    ↓
EventClassifier → {category: "military", weight: 0.9}
    ↓
Feature: military_event_weight = 0.9
    ↓
TFT Input: [..., military=0.9, sentiment=-0.8, ...]
    ↓
TFT Output: price↑ (لأن أحداث عسكرية ← عدم استقرار ← ارتفاع دولار)
    ↓
Dashboard: "⚠️ توقع ارتفاع السعر إلى 14800 خلال 24 ساعة"
```

```
كشط ويب: سعر جديد 14500
    ↓
InfluxDB: write_price("exchange_rate", 14500)
    ↓
Feature: syp_usd_price = 14500, price_change = +1.2%
    ↓
TFT Input: [..., price=14500, change=0.012, ...]
    ↓
TFT Output: الكميات [14200, 14400, 14600, 14800, 15100]
```

```
API خارجي: سعر الذهب ارتفع 3%
    ↓
Feature: gold_price_usd = 2050
    ↓
```
