# التقنيات المستخدمة — مشروع ShamIn

## نظرة عامة

مشروع ShamIn يستخدم مجموعة متكاملة من التقنيات الحديثة لبناء نظام تنبؤ بسعر صرف الليرة السورية مقابل الدولار. هذا الملف يشرح كل تقنية، ولماذا اختيرت، وكيف تُستخدم.

---

## 1. لغة البرمجة

### Python 3.11+
- **ما هي**: لغة برمجة عالية المستوى، سهلة القراءة والكتابة
- **لماذا اخترناها**: الأغنى في مكتبات الذكاء الاصطناعي ومعالجة اللغة الطبيعية. مجتمع ضخم ودعم واسع
- **أين تُستخدم**: كل شيء في المشروع مكتوب بـ Python
- **الإصدار**: 3.11 أو أحدث (لدعم `asyncio.TaskGroup` و Performance Improvements)

---

## 2. التعلم العميق والتنبؤ

### PyTorch 2.3+
- **ما هي**: مكتبة تعلم عميق مفتوحة المصدر من Meta (Facebook)
- **لماذا اخترناها**: أسرع في التطوير والتجريب من TensorFlow. تدعم CUDA للتسريع على GPU
- **أين تُستخدم**: بناء نموذج LSTM للمشاعر، ونموذج TFT الرئيسي
- **ملف**: `src/models/` (سيُنفذ في المراحل 5-6)

### PyTorch Lightning
- **ما هي**: إطار عمل فوق PyTorch يُنظم كود التدريب
- **لماذا اخترناها**: يُبسط حلقة التدريب (Training Loop)، يدعم التدريب الموزع، يُسهل تتبع التجارب
- **أين تُستخدم**: تدريب نموذج TFT
- **ملف**: `src/training/` (سيُنفذ في المرحلة 6)

### PyTorch Forecasting
- **ما هي**: مكتبة متخصصة في التنبؤ بالسلاسل الزمنية فوق PyTorch Lightning
- **لماذا اخترناها**: تُوفر تنفيذاً جاهزاً لنموذج Temporal Fusion Transformer (TFT) — أحدث نماذج التنبؤ
- **أين تُستخدم**: النموذج الرئيسي للتنبؤ بالسعر
- **ملف**: `src/models/tft_model.py` (سيُنفذ في المرحلة 6)

### نموذج TFT (Temporal Fusion Transformer)
- **ما هو**: نموذج تعلم عميق صُمم خصيصاً للتنبؤ بالسلاسل الزمنية المتعددة المتغيرات
- **لماذا اخترناه**: يتعامل مع البيانات المفقودة، يُعطي تفسيراً للتنبؤ (أي ميزة أثرت أكثر)، يُخرج كميات (Quantiles) وليس فقط نقطة واحدة
- **الإعدادات** (من `config/model_config.yaml`):
  - `hidden_size: 128` — حجم الطبقة المخفية
  - `lstm_layers: 2` — عدد طبقات LSTM
  - `attention_heads: 4` — عدد رؤوس الانتباه
  - `dropout: 0.3` — نسبة الإسقاط
  - `quantiles: [0.05, 0.25, 0.5, 0.75, 0.95]` — الكميات المتنبأ بها

## 3. التعلم الآلي التقليدي

### XGBoost
- **ما هي**: مكتبة تعلم آلي تعتمد على Gradient Boosted Trees
- **لماذا اخترناها**: سريعة جداً في التدريب والاستدلال، ممتازة للبيانات الجدولية، تعمل كنموذج أساسي (Baseline) للمقارنة
- **أين تُستخدم**: نموذج مساعد للمقارنة وفي نظام التجميع (Ensemble)
- **الإعدادات**: `n_estimators: 500`, `max_depth: 8`, `learning_rate: 0.05`

### statsmodels (Holt-Winters / ARIMA)
- **ما هي**: مكتبة إحصائية تقليدية للتنبؤ
- **لماذا اخترناها**: نموذج Holt-Winters يلتقط الأنماط الموسمية. يعمل كنموذج أساسي (Baseline) بسيط
- **أين تُستخدم**: نموذج إحصائي ضمن التجميع
- **الإعدادات**: `seasonal: True`, `seasonal_periods: 24` (دورة يومية)

### scikit-learn
- **ما هي**: المكتبة الأشمل للتعلم الآلي التقليدي
- **لماذا اخترناها**: أدوات التقييم والتحويل والتقسيم والمعالجة المسبقة
- **أين تُستخدم**: تقسيم البيانات، حساب المقاييس (MAE, RMSE, R²)، تحجيم الميزات

---

## 4. معالجة اللغة العربية (Arabic NLP)

### CAMeL Tools
- **ما هي**: أدوات من جامعة نيويورك أبوظبي لمعالجة اللغة العربية
- **لماذا اخترناها**: الأدق في تحليل الصرف العربي، تدعم لهجات متعددة
- **أين تُستخدم**: تحليل المشاعر وتصنيف الأحداث (المرحلة 3-4)

### Farasa
- **ما هي**: أداة سعودية لتقطيع النص العربي إلى جذور وكلمات فرعية
- **لماذا اخترناها**: أسرع مُقطِّع (Segmenter) عربي متاح
- **أين تُستخدم**: `src/processing/text/tokenizer.py` — تقطيع اختياري (إذا كانت المكتبة مُثبتة)

### arabic-reshaper + python-bidi
- **ما هي**: مكتبات لإصلاح عرض النص العربي من اليمين لليسار
- **أين تُستخدم**: عرض النصوص العربية في الرسوم البيانية والتقارير

---

## 5. جمع البيانات

### Telethon
- **ما هي**: عميل Telegram API غير رسمي مبني على asyncio
- **لماذا اخترناها**: تتيح قراءة رسائل قنوات تلغرام programmatically. تدعم حسابات متعددة وتدوير الجلسات
- **أين تُستخدم**: `src/ingestion/collectors/telegram_collector.py`
- **الميزات المستخدمة**: `client.iter_messages()`, `FloodWaitError` handling, session pooling

### feedparser
- **ما هي**: مكتبة لتحليل خلاصات RSS/Atom
- **لماذا اخترناها**: بسيطة وموثوقة لقراءة أخبار RSS
- **أين تُستخدم**: `src/ingestion/collectors/rss_collector.py`

### BeautifulSoup4 (bs4)
- **ما هي**: مكتبة لتحليل واستخراج بيانات من HTML/XML
- **لماذا اخترناها**: سهلة الاستخدام مع CSS Selectors
- **أين تُستخدم**: `src/ingestion/collectors/web_scraper.py`

### aiohttp
- **ما هي**: عميل/خادم HTTP غير متزامن (Async)
- **لماذا اخترناها**: أسرع من `requests` للطلبات المتزامنة. تعمل مع asyncio
- **أين تُستخدم**: إرسال طلبات HTTP في الكاشط وجامع API

### Selenium (اختياري)
- **ما هي**: أداة أتمتة المتصفح
- **لماذا**: لصفحات تُحمَّل بـ JavaScript ولا يمكن كشطها بـ aiohttp
- **أين تُستخدم**: كشط مواقع أسعار معينة تستخدم JavaScript

---

## 6. قواعد البيانات والتخزين

### PostgreSQL 16
- **ما هي**: قاعدة بيانات علائقية مفتوحة المصدر (الأقوى)
- **لماذا اخترناها**: تدعم JSONB (تخزين JSON مع فهارس)، أداء ممتاز، موثوقة جداً
- **أين تُستخدم**: تخزين الأخبار والتنبؤات وأداء النماذج (7 جداول)
- **Docker**: `shamin_postgres` على المنفذ 5432
- **الجداول**: `sources`, `raw_news`, `processed_news`, `predictions`, `model_performance`, `classified_events`, `drift_monitoring`

### InfluxDB 2.7
- **ما هي**: قاعدة بيانات متخصصة في السلاسل الزمنية (Time-Series Database)
- **لماذا اخترناها**: مُصممة خصيصاً لتخزين بيانات مُختمة بالوقت (أسعار كل 5 دقائق). ضغط ممتاز وأداء عالٍ
- **أين تُستخدم**: تخزين بيانات أسعار الصرف اللحظية
- **Docker**: `shamin_influxdb` على المنفذ 8086
- **المفاهيم**: Bucket (دلو) = مجموعة بيانات، Measurement = نوع القياس، Tag = علامة، Field = قيمة

### Redis 7
- **ما هي**: قاعدة بيانات في الذاكرة (In-Memory) بسرعة فائقة
- **لماذا اخترناها**: وسيط رسائل (Broker) لـ Celery، تخزين مؤقت (Cache)
- **أين تُستخدم**: وسيط بين Celery Beat (الجدولة) و Celery Worker (التنفيذ)
- **Docker**: `shamin_redis` على المنفذ 6379

### MinIO
- **ما هي**: خادم تخزين كائنات متوافق مع Amazon S3
- **لماذا اخترناها**: تخزين ملفات كبيرة (نماذج مُدربة، datasets) بتكلفة صفر (مفتوحة المصدر)
- **أين تُستخدم**: `src/storage/object_store.py` — حفظ وتحميل النماذج المُدربة
- **Docker**: `shamin_minio` على المنفذ 9000 (API) و 9001 (Console)

---

## 7. ORMs وإدارة البيانات

### SQLAlchemy 2.0
- **ما هي**: ORM (Object-Relational Mapping) لـ Python — يُتيح التعامل مع قواعد البيانات بالكائنات بدل SQL
- **لماذا اخترناها**: الأنضج والأقوى. تدعم connection pooling وmigration
- **أين تُستخدم**: `scripts/setup_db.py` (تعريف الجداول)، `src/storage/relational_db.py` (الاتصال)
- **المفاهيم**: `Engine` = محرك الاتصال، `Session` = جلسة عمل، `Model` = كلاس يمثل جدول

### Pydantic 2.0
- **ما هي**: مكتبة للتحقق من البيانات (Data Validation) باستخدام Type Hints
- **لماذا اخترناها**: تضمن صحة البيانات تلقائياً. مُدمجة مع FastAPI
- **أين تُستخدم**: `src/ingestion/raw_schema.py` — تعريف `RawDataItem` و `CollectionResult`

### Alembic
- **ما هي**: أداة هجرة قواعد البيانات (Database Migration) تعمل مع SQLAlchemy
- **لماذا اخترناها**: تتبُّع نسخ البنية (Schema Versioning). تُتيح الترقية والتراجع
- **أين تُستخدم**: إدارة تعديلات بنية قاعدة البيانات

---

## 8. المهام غير المتزامنة والجدولة

### Celery
- **ما هي**: إطار عمل لتنفيذ المهام الموزعة في الخلفية
- **لماذا اخترناها**: تُتيح جدولة مهام الجمع (كل 5 أو 15 دقيقة) وموازنة التنفيذ
- **أين تُستخدم**: `src/ingestion/scheduler.py`
- **المكونات**:
  - **Celery Worker**: يُنفذ المهام
  - **Celery Beat**: يُرسل المهام حسب الجدول الزمني
  - **Broker** (Redis): الوسيط بين Beat و Worker
- **Docker**: `shamin_celery_worker` و `shamin_celery_beat`

### asyncio
- **ما هي**: مكتبة Python المدمجة للبرمجة غير المتزامنة
- **لماذا اخترناها**: تُتيح تشغيل عمليات I/O (شبكة، قراءة ملفات) بالتوازي دون خيوط
- **أين تُستخدم**: جميع الجامعات (collectors) تستخدم `async/await`

## 9. واجهات التطبيق

### FastAPI
- **ما هي**: إطار عمل حديث لبناء REST APIs بـ Python
- **لماذا اخترناها**: الأسرع أداءً (مبني على Starlette + Uvicorn). توليد توثيق تلقائي (Swagger)
- **أين تُستخدم**: `src/presentation/api/main.py` — نقاط النهاية للتنبؤات والصحة
- **الميزات**: CORS مُفعَّل، دعم async، Pydantic integration

### Uvicorn
- **ما هي**: خادم ASGI عالي الأداء
- **لماذا اخترناها**: الخادم القياسي لـ FastAPI
- **أين تُستخدم**: تشغيل تطبيق FastAPI

### Streamlit
- **ما هي**: إطار عمل لبناء لوحات بيانات تفاعلية بسرعة
- **لماذا اخترناها**: تُحوِّل سكريبت Python إلى تطبيق ويب بـ 10 أسطر. مثالية للوحات البيانات
- **أين تُستخدم**: `src/presentation/dashboard/app.py` — لوحة المراقبة الرئيسية

### Plotly
- **ما هي**: مكتبة رسوم بيانية تفاعلية
- **لماذا اخترناها**: رسوم بيانية جميلة وتفاعلية (zoom, hover, export). تتكامل مع Streamlit
- **أين تُستخدم**: عرض مخططات الأسعار والتنبؤات

## 10. تحسين وتتبع النماذج

### Optuna
- **ما هي**: إطار عمل لتحسين المعاملات الفائقة (Hyperparameter Optimization)
- **لماذا اخترناها**: خوارزمية TPE (Tree-structured Parzen Estimator) ذكية — تتعلم من التجارب السابقة
- **أين تُستخدم**: البحث عن أفضل إعدادات النماذج (المرحلة 7)

### MLflow
- **ما هي**: منصة مفتوحة المصدر لإدارة دورة حياة التعلم الآلي
- **لماذا اخترناها**: تتبُّع التجارب، مقارنة النماذج، تسجيل المقاييس
- **أين تُستخدم**: تسجيل كل تجربة تدريب مع المعاملات والنتائج

### SHAP (SHapley Additive exPlanations)
- **ما هي**: مكتبة لتفسير تنبؤات النماذج
- **لماذا اخترناها**: تُحدد أي ميزة أثرت في التنبؤ وبأي مقدار
- **أين تُستخدم**: تفسير تنبؤات TFT — لماذا توقع النموذج هذا السعر؟

## 11. الحاويات والنشر

### Docker
- **ما هو**: منصة حاويات لتعبئة التطبيقات مع كل اعتمادياتها
- **لماذا اخترناه**: يضمن أن البيئة متطابقة في التطوير والإنتاج
- **أين يُستخدم**: `Dockerfile` — بناء صورة التطبيق

### Docker Compose
- **ما هو**: أداة لتعريف وتشغيل عدة حاويات معاً
- **لماذا اخترناه**: تشغيل كل الخدمات الست بأمر واحد: `docker-compose up`
- **أين يُستخدم**: `docker-compose.yml` — يُعرِّف 6 خدمات:
  1. `shamin_postgres` — قاعدة PostgreSQL
  2. `shamin_influxdb` — قاعدة InfluxDB
  3. `shamin_redis` — وسيط Redis
  4. `shamin_minio` — تخزين MinIO
  5. `shamin_celery_worker` — مُنفذ المهام
  6. `shamin_celery_beat` — مُجدوِل المهام
