# دليل التشغيل — إعداد وتشغيل مشروع ShamIn خطوة بخطوة

## المتطلبات الأساسية

قبل البدء، تأكد من تثبيت:

| البرنامج | الإصدار المطلوب | رابط التحميل | الوصف |
|---------|----------------|-------------|-------|
| Python | 3.11 أو أحدث | python.org | لغة البرمجة |
| Docker Desktop | أحدث إصدار | docker.com | لتشغيل الخدمات |
| Git | أي إصدار | git-scm.com | لإدارة الكود |

---

## الخطوة 1: تحميل المشروع

```bash
# استنساخ المستودع
git clone <repository-url>
cd ShamIn
```

---

## الخطوة 2: إنشاء البيئة الافتراضية

```bash
# إنشاء بيئة افتراضية
python -m venv venv

# تفعيل البيئة (Windows)
venv\Scripts\activate

# تفعيل البيئة (Linux/Mac)
source venv/bin/activate
```

**لماذا البيئة الافتراضية؟** تمنع تعارض المكتبات بين المشاريع. كل مشروع له مكتبات منفصلة.

---

## الخطوة 3: تثبيت المكتبات

```bash
pip install -r requirements.txt
```

**ملاحظة**: بعض المكتبات (مثل `camel-tools` و `farasapy`) قد تحتاج خطوات إضافية:

```bash
# تثبيت بيانات CAMeL Tools (اختياري — للمرحلة 3+)
camel_data -i disambig-mle-calima-msa-r13

# تثبيت Farasa (اختياري)
pip install farasapy
```

## الخطوة 4: إعداد متغيرات البيئة

```bash
# انسخ ملف المثال
cp .env.example .env

# أو على Windows
copy .env.example .env
```

**ثم عدِّل `.env` بالقيم الحقيقية**:

```env
# قاعدة PostgreSQL
POSTGRES_URL=postgresql://shamin:shamin_pass_2024@localhost:5432/shamin_db
POSTGRES_USER=shamin
POSTGRES_PASSWORD=shamin_pass_2024
POSTGRES_DB=shamin_db

# InfluxDB
INFLUXDB_URL=http://localhost:8086
INFLUXDB_TOKEN=shamin-dev-token
INFLUXDB_ORG=shamin_org
INFLUXDB_BUCKET=exchange_rates
INFLUXDB_ADMIN_USER=admin
INFLUXDB_ADMIN_PASSWORD=shamin_influx_2024

# Redis
REDIS_URL=redis://localhost:6379/0

# MinIO
MINIO_ENDPOINT=localhost:9000
MINIO_ACCESS_KEY=shamin_minio
MINIO_SECRET_KEY=shamin_minio_secret

# Telegram (احصل عليها من https://my.telegram.org)
TELEGRAM_API_ID=<معرفك>
TELEGRAM_API_HASH=<بصمتك>
TELEGRAM_PHONE=<رقم هاتفك>
```

### كيف تحصل على بيانات Telegram:
1. ادخل إلى https://my.telegram.org
2. سجل دخول برقم هاتفك
3. اختر "API Development Tools"
4. أنشئ تطبيقاً جديداً
5. انسخ `api_id` و `api_hash`

## الخطوة 5: تشغيل الخدمات بـ Docker

```bash
# تشغيل جميع الخدمات في الخلفية
docker-compose up -d
```

**ما يحدث**: Docker يُنشئ ويُشغل 6 حاويات:

| الحاوية | المنفذ | الوصف |
|---------|-------|-------|
| `shamin_postgres` | 5432 | قاعدة PostgreSQL |
| `shamin_influxdb` | 8086 | قاعدة InfluxDB |
| `shamin_redis` | 6379 | وسيط Redis |
| `shamin_minio` | 9000, 9001 | تخزين MinIO |
| `shamin_celery_worker` | — | مُنفذ المهام |
| `shamin_celery_beat` | — | مُجدوِل المهام |

### التحقق من التشغيل:

```bash
# عرض حالة الحاويات
docker-compose ps

# يجب أن تظهر جميعها بحالة "Up"
```

### الوصول لواجهات الإدارة:

| الخدمة | الرابط | المستخدم | كلمة المرور |
|--------|--------|---------|------------|
| MinIO Console | http://localhost:9001 | `shamin_minio` | `shamin_minio_secret` |
| InfluxDB UI | http://localhost:8086 | `admin` | `shamin_influx_2024` |

## الخطوة 6: إنشاء جداول قاعدة البيانات

```bash
python scripts/setup_db.py
```

**ما يحدث**: يُنشئ 7 جداول في PostgreSQL:
1. `sources` — المصادر
2. `raw_news` — الأخبار الخام
3. `processed_news` — الأخبار المعالجة
4. `predictions` — التنبؤات
5. `model_performance` — أداء النماذج
6. `classified_events` — الأحداث المصنفة
7. `drift_monitoring` — مراقبة الانجراف

### التحقق من الجداول:

```bash
# الاتصال بقاعدة البيانات
docker exec -it shamin_postgres psql -U shamin -d shamin_db

# عرض الجداول
\dt

# الخروج
\q
```

## أوامر مفيدة

### إدارة Docker:

```bash
# إيقاف جميع الخدمات
docker-compose down

# إعادة تشغيل خدمة معينة
docker-compose restart shamin_postgres

# عرض سجلات خدمة
docker-compose logs -f shamin_celery_worker

# حذف كل شيء (تحذير: يحذف البيانات!)
docker-compose down -v
```

### إدارة Celery:

```bash
# تشغيل Worker يدوياً (بدون Docker)
celery -A src.ingestion.scheduler worker --loglevel=info

# تشغيل Beat يدوياً
celery -A src.ingestion.scheduler beat --loglevel=info

# فحص المهام المُسجلة
celery -A src.ingestion.scheduler inspect registered
```

### إدارة قاعدة البيانات:

```bash
# نسخ احتياطي
docker exec shamin_postgres pg_dump -U shamin shamin_db > backup.sql

# استعادة
docker exec -i shamin_postgres psql -U shamin shamin_db < backup.sql
```

## حل المشاكل الشائعة

### المشكلة: خطأ اتصال بـ PostgreSQL
```
sqlalchemy.exc.OperationalError: could not connect to server
```
**الحل**: تأكد أن حاوية PostgreSQL تعمل:
```bash
docker-compose ps shamin_postgres
docker-compose up -d shamin_postgres
```

### المشكلة: FloodWaitError في Telegram
```
telethon.errors.FloodWaitError: A wait of 300 seconds is required
```
**الحل**: النظام يتعامل مع هذا تلقائياً. للتجنب: أضف حسابات Telegram متعددة في الإعدادات

### المشكلة: خطأ في تثبيت torch
```
ERROR: No matching distribution found for torch>=2.3.0
```
**الحل**:
```bash
# Windows مع CUDA
pip install torch --index-url https://download.pytorch.org/whl/cu121

# بدون GPU
pip install torch --index-url https://download.pytorch.org/whl/cpu
```

### المشكلة: منفذ مشغول
```
Error: port 5432 already in use
```
**الحل**: غيِّر المنفذ في `docker-compose.yml`:
```yaml
ports:
  - "5433:5432"   # استخدم 5433 بدلاً من 5432
```

### المشكلة: نفاد ذاكرة GPU
```
RuntimeError: CUDA out of memory
```
**الحل**: النظام يتعامل تلقائياً عبر `batch_size_estimator.py` — يُقلل حجم الدفعة

## هيكل التشغيل الكامل

```
1. docker-compose up -d          ← تشغيل الخدمات
2. python scripts/setup_db.py    ← إنشاء الجداول
3. python scripts/setup_influxdb.py ← إعداد InfluxDB
4. python scripts/setup_minio.py    ← إعداد MinIO
5. uvicorn ... --port 8000       ← تشغيل API
6. streamlit run ... --port 8501 ← تشغيل Dashboard
7. الجمع يعمل تلقائياً عبر Celery ← كل 5-15 دقيقة
```
